What it is

A comic reader built for the terminal. Panels appear in ASCII boxes, arrows turn the page. It feels like flipping through a photocopied mini-comic pulled from a spinner rack.

Comics are parsed from simple text files with ASCII art and dialogue. The reader handles panel layouts, page turns, and even supports speech bubbles and sound effects rendered in monospace glory.

Run it

Terminal
$ git clone https://github.com/tinytuis/tinytuis.git
$ cd tinytuis
$ python issues/gutter/main.py

Requires Python 3.7+ and the blessed library.

Notes

Inspired by the DIY comic scene—zines, mini-comics, and the raw energy of photocopied art. What if comics could live in the terminal, rendered in the same monospace font as code?

The comic format is intentionally simple: panels defined by ASCII borders, dialogue in quotes, and basic layout instructions. Anyone can write a comic in a text editor.

Supports multi-panel layouts, splash pages, and even animated sequences. The gutter—the space between panels—becomes part of the terminal's natural grid system.

Technical Details

Language: Python 3.7+
Dependencies: blessed, threading
Theme: Minimal (grays and amber)
Input: Arrow keys, space, 'f', 'q' to quit
